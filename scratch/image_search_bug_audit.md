# 이미지 검색 아키텍처 결함 조사 보고서 (Image Search Bug Audit)

**조사 일시**: 2026-04-24
**대상 현상**: 비전 인식 결과(맥미니)와 실제 검색 쿼리(아이패드) 불일치로 인한 결과 0건 버그

---

## 1. 이미지 검색 파이프라인 분석 (`js/thisone_app_v3_final.js`)

현재 `sendMsg` 함수 내 이미지 검색 처리 로직은 다음과 같은 **2단계 구조**로 되어 있습니다.

### [Phase 1: 검색용 의도 추론] (298~311행)
- `window.ThisOneAPI.requestIntentInfer`를 호출하여 이미지 분석을 시도합니다.
- 성공 시 `intentProfile.refinedSearchTerm`을 `finalSearchQuery`로 설정합니다.
- **결함**: 만약 이 과정에서 네트워크 타임아웃이나 AI 엔진(Gemini) 503 오류가 발생하면, `catch` 블록(308행)으로 넘어가고 **`finalSearchQuery`는 수정되지 않은 채 기존 텍스트 쿼리(`queryText`)를 유지**합니다.

### [Phase 2: 검색 실행] (313행)
- `finalSearchQuery`로 네이버 API 검색을 수행합니다.
- Phase 1이 실패했다면, 사용자가 이전에 입력했거나 남아있던 텍스트(예: "아이패드 에어 M2")로 검색이 실행됩니다.

### [Phase 3: 랭킹용 의도 재추론] (340~346행)
- `intentProfile`이 null일 경우(Phase 1 실패 시), **다시 한 번** `requestIntentInfer`를 호출하여 결과를 받아옵니다.
- **확정 원인**: 이때 호출은 성공하여 `intentProfile`이 "애플 맥 미니 M2"로 올바르게 설정됩니다.

---

## 2. 버그 발생 시나리오 (확정 원인)

1. 사용자가 이전에 "아이패드 에어 M2"를 검색했거나 입력창에 해당 텍스트가 남아있는 상태에서 "맥미니" 이미지를 업로드함.
2. `sendMsg` 시작: `queryText`는 "아이패드 에어 M2"로 설정됨.
3. **1차 분석 실패**: `intentInfer` 호출이 일시적 오류(503 등)로 실패. `finalSearchQuery`는 여전히 "아이패드 에어 M2".
4. **잘못된 검색**: 네이버 쇼핑에서 "아이패드 에어 M2" 결과들을 가져옴.
5. **2차 분석 성공**: 342행에서 다시 호출한 `intentInfer`는 성공하여 "맥미니" 정보를 가져옴.
6. **모순 발생**: 랭킹 AI(Gemini/GPT)는 "이미지는 맥미니인데, 검색 결과는 온통 아이패드뿐"인 상황을 인지함.
7. **전량 배제**: 랭킹 AI가 지능적으로 "이건 맥미니가 아니라 아이패드이므로 제외"라고 판정하여 결과가 0건이 됨.

---

## 3. 관련 변수 및 연결 고리

- `currentQuery`: `msgInput`의 값을 저장하는 전역 변수. 이미지 검색 시에도 빈 값이 아니면 `queryText`로 사용됨 (262행).
- `finalSearchQuery`: 실제 네이버 검색에 쓰이는 변수. `intentInfer` 성공 시에만 이미지 기반으로 업데이트됨.
- `intentProfile`: 랭킹 AI와 UI에 전달되는 의도 객체. 검색 실행 시점과 랭킹 시점에 두 번 결정될 수 있는 구조임.

---

## 4. 가설 검증 방법 (오너 재현용)

1. 브라우저에서 "아이패드"를 먼저 검색하여 입력창과 `currentQuery`를 오염시킴.
2. `js/api.js`의 `requestIntentInfer` 함수 시작 부분에 일시적으로 `throw new Error()`를 심어 1차 호출을 강제 실패시킴.
3. 맥미니 이미지를 업로드하여 검색 실행.
4. 현상 재현 확인: 검색은 아이패드로 되고, 제외 사유는 맥미니로 나옴.

---

## 5. 수정 방안 (제안)

1. **중복 호출 제거**: `requestIntentInfer`는 한 번만 호출하고, 그 결과를 끝까지 공유해야 함.
2. **Fail-Fast 전략**: 이미지 검색 시 1차 의도 추론이 실패하면 잘못된 텍스트 쿼리로 검색을 강행하지 말고, 사용자에게 "이미지 분석에 실패했습니다"라고 즉시 알리거나 재시도를 유도해야 함.
3. **입력창 자동 정리**: 이미지 업로드 시 기존 텍스트 쿼리와의 충돌을 방지하기 위해 이미지 업로드 즉시 입력창을 초기화하는 로직 추가 필요.
