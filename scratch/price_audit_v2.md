## 1. 중앙값 기반 배제 로직 조사 결과

### [존재 여부] 
- **확인**: `ranking.js` 내에 중앙값 계산 및 배제 로직이 **존재함**.
  - `getMedianPrice(items)` (187-202행): 가격이 0보다 큰 아이템들을 정렬하여 중앙값 산출.
  - `shouldExcludeFromPriceRank` (501-513행): 중앙값의 30% 미만인 상품을 `exclude: true` 처리.

### [실패 원인 분석]
1. **중앙값 오염 (Median Poisoning)**: 
   - `getMedianPrice`는 전처리 없이 모든 `priceNum > 0`인 항목을 계산에 포함함.
   - 50개 검색 결과 중 1원, 10원, 120원 등 미끼 상품이 다수 포함될 경우, 중앙값 자체가 비정상적으로 낮아짐.
   - 예: 120원이 다수일 경우 중앙값이 120원 근처로 형성되어, 120원 본인 및 그 이상의 소모품들이 필터를 통과하게 됨.
2. **SafetyNet에 의한 복원 (Case B)**:
   - `buildCandidates` (753-764행): 필터링 후 남은 본품이 3개 미만일 경우, **제거했던 모든 항목을 다시 복원**하여 보여줌. 
   - 아이패드 1원 등이 필터링되더라도, 남은 본품이 없으면 다시 화면에 노출됨.

## 2. 극단값 통과 가설 검증

- **가설 A (중앙값 포함)**: **참**. 1원 상품이 중앙값 계산 배열에 포함됨.
- **가설 B (필터 미작동)**: **참 (복합적)**. 플래그는 달리지만 `SafetyNet` 로직에 의해 결과 배열에 다시 추가됨.
- **가설 C (필터 부재)**: **거짓**. 필터는 존재하나 설계 결함(오염 방지 부재)과 예외 로직(SafetyNet)에 의해 무력화됨.

## 3. 키워드 감지 로직 존재 여부 (렌탈/약정)

- **현황**: `isRentalOrContract` (140-150행) 존재.
- **감지 키워드**: `대여`, `렌탈`, `약정`, `가입`, `요금제`, `통신사`, `월납`, `공시지원`.
- **누락 키워드**: **할부**, **대리점**, **조건부**, **프로모션**, **신규가입(일부)**, **번호이동**, **기기변경**.
- **한계**: 가격이 1000원 이하일 때만 의심(`isSuspiciousPrice`)하는 로직이 있어, 1200원~5000원 사이의 미끼 상품은 통과될 가능성 높음.

## 4. 네이버 Shopping API lprice 신뢰성 (마우스패드)

- **조사 결과**: 네이버 쇼핑 웹 검색 시 CAPTCHA 차단으로 실시간 비교는 제한적이나, 통상적인 API 특성 확인.
- **분석**:
  - API `lprice`는 해당 상품의 **최저가 옵션**을 반환함.
  - 마우스패드 상품 페이지 내에 '스티커'나 '청소포'가 120원 옵션으로 있으면, API는 이를 마우스패드의 가격으로 보고함.
  - **결론**: API 반환값 120원은 허수(Bait)이며, 시스템이 이를 실제 마우스패드 본품 가격과 구분하지 못하고 있음.

## 종합 판정 및 제안
현재 시스템은 **"데이터 오염에 취약한 중앙값 필터"**와 **"본품 부재 시 미끼 상품을 강제 복원하는 안전장치"**가 충돌하여 필터링 효과가 0에 수렴하고 있습니다.

**차기 시도 제안:**
1. 중앙값 계산 전 상/하위 20%를 잘라내는 **Trimming** 로직 도입.
2. 카테고리별 **최소 가격 하한선(Hard Floor)** 하드코딩 도입.
3. `SafetyNet` 작동 시 소모품이 아닌 **AI 추천 대체 문구** 노출로 변경.
