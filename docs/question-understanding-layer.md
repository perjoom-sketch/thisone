# ThisOne 질문 이해·추가 조사 레이어

## 1. 왜 이 레이어가 필요한가

ThisOne은 사용자의 원문 질문을 곧바로 검색어로 쓰면 질문을 너무 문자 그대로 해석할 수 있다. 사용자는 늘 검색에 적합한 표현으로 묻지 않으며, 짧은 말·구어체·생략된 맥락·기관명과 역할 단어가 섞인 질문이 자주 들어온다.

따라서 즉답 또는 각 모드의 최종 답변을 만들기 전에 공통 레이어가 먼저 질문을 구조화한다. 이 레이어는 답변을 생성하지 않고, 모드별 답변기가 더 좋은 검색과 근거 확인을 할 수 있도록 분석 결과를 제공한다.

핵심 원칙은 다음과 같다.

> 질문이 좋아야 답이 좋아진다.

> 디스원은 사용자의 원문 질문을 그대로 검색하지 않는다. 먼저 사용자의 의도를 파악해 더 정확한 내부 질문으로 재작성한 뒤, 그 질문을 기준으로 검색·출처 확인·답변을 수행한다.

## 2. ThisOne 처리 흐름

```text
raw user input
→ intent analysis
→ internal rewritten question
→ evidence preference
→ search query generation
→ deeper research if needed
→ interim report
→ mode-specific answer
```

각 단계의 역할은 다음과 같다.

1. **raw user input**: 사용자의 원문 질문을 받는다.
2. **intent analysis**: 질문이 소속/지위 확인인지, 방법 안내인지, 추천인지, 오류 해결인지 같은 넓은 작업 유형을 판정한다.
3. **internal rewritten question**: 검색과 출처 확인에 더 적합한 내부 질문으로 바꾼다.
4. **evidence preference**: 공식 출처, 일반 웹, 사용자 제공 자료, 일반 지식 중 어떤 근거가 우선인지 정한다.
5. **search query generation**: 원문 문장 하나가 아니라 키워드형 검색어 여러 개를 만든다.
6. **deeper research if needed**: 첫 검색에서 충분한 출처가 없으면 즉시 답변을 끝내지 않고 더 깊은 조사 쿼리로 전환한다.
7. **interim report**: 추가 조사로 넘어갈 때만 짧은 한국어 상태 메시지를 제공한다.
8. **mode-specific answer**: 즉답, 해석, 럽미, 집밥, 쇼핑, 서치 등 각 모드가 자기 톤과 정책에 맞게 최종 답변을 만든다.

## 3. 공통 분석 결과

`lib/questionUnderstanding.js`의 `analyzeQuestion(input, options)`는 규칙 기반으로 다음 정보를 반환한다.

- 원문과 정규화된 질문
- 언어 판정
- 의도 유형과 작업 유형
- 내부 재작성 질문
- 엔티티, 핵심 문구, 역할 단어, 기관 단어
- 검색 필요 여부, 최신 정보 필요 여부, 공식 출처 필요 여부
- 근거 선호도
- 1차 검색 쿼리와 추가 조사 쿼리
- 답변 전략
- 추가 조사 전환 시 보여줄 짧은 중간보고 메시지

이 레이어는 **AI 모델을 호출하지 않는다.** 이번 단계에서는 규칙 기반 공통 기반만 만들고, 최종 답변 생성은 각 모드에 남긴다.

## 4. 추가 조사 원칙

> 정보 부족은 답변 종료 사유가 아니라 추가 조사 전환 조건이다.

첫 검색에서 유용한 출처가 부족하면 ThisOne은 곧바로 “현재 정보로는 알 수 없습니다.”라고 끝내지 않는다. 대신 다음을 수행한다.

1. 질문을 더 정확한 검색 질문으로 다시 정리한다.
2. 공식·공공·법령·기관 자료가 필요한 질문이면 공식 출처 중심으로 다시 찾는다.
3. 역할/소속/권한/대상/기한 등 확인되지 않은 부분을 분리한다.
4. 그래도 공개 출처가 충분하지 않으면 확인된 범위, 합리적으로 추정 가능한 범위, 사용자가 다음에 확인할 지점을 나눠서 답한다.

추가 조사 전환 시 사용할 수 있는 짧은 메시지는 다음과 같다.

- 질문 의도를 파악하고 있습니다.
- 검색 질문을 다시 정리하고 있습니다.
- 공식 출처와 관련 자료를 확인하고 있습니다.
- 근거가 부족한 부분을 분리해 정리하고 있습니다.

이 메시지는 정상적인 모든 검색에 반복 노출하지 않고, 1차 검색에서 근거가 약해 추가 조사로 넘어갈 때만 사용한다.

## 5. 이후 적용 대상 모드

이 레이어는 다음 모드가 순차적으로 공유할 수 있다.

- 즉답
- 해석
- 럽미
- 집밥
- 쇼핑
- 서치

이번 PR에서는 **즉답만 최소 통합**한다. 해석, 럽미, 집밥, 쇼핑, 서치의 최종 답변 방식과 톤은 변경하지 않는다.

## 6. 금지 사항

이 문제를 수백만 개의 주제별 특수 처리로 해결하면 안 된다.

- 특정 답변을 하드코딩하지 않는다.
- 특정 한 주제만 겨냥한 예외 규칙을 만들지 않는다.
- 가짜 출처를 만들지 않는다.
- 원문 Serper JSON을 사용자에게 노출하지 않는다.
- 쇼핑 랭킹, 해석 동작, 럽미 톤, 집밥 안전 규칙, analytics, KV는 이 레이어 도입과 별개로 유지한다.

질문 이해 레이어는 계속 범용 규칙이어야 하며, 모드별 답변기는 이 구조화 결과를 받아 각자의 책임 범위에서 답한다.
