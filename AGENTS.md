## 1. 절대 규칙 (영구 · 바꾸지 말 것)
==============================================

### 1-1. 레이아웃 원칙
- 모든 레이아웃은 목록형(행 나열) 구조로 통일
- PC/모바일 동일 구조, 폭만 반응형으로 변화
- 카드 그리드 구조 금지 (쿠팡/네이버쇼핑 스타일 목록형 유지)

### 1-2. 반응형 원칙
- 고정 px 너비(width, max-width) 금지
- 모든 치수는 `clamp()` / `min()` / `max()` / `vw` 기반
- 모바일 브레이크포인트: **640px** (아이폰 12에서 검증됨)

### 1-3. 디자인 톤 원칙
- 메인 카피: 과도한 폰트 크기/굵기/검정색 금지
- 브랜드 톤: 블루 포인트 + 라이트 그레이 베이스
- 배지/보조 UI 요소는 기능과 시각적으로 연결되어야 함
  (떨어진 위치 = 의미 전달 실패)

### 1-4. 사용 LLM 모델 (이중화 체계)
- **Primary (1순위)**: gemini-2.5-flash
  - API 버전: v1 (안정 엔드포인트)
  - 연동: Google Generative AI SDK
- **Secondary (2순위 폴백)**: gpt-5.4-mini
  - 역할: Gemini 503(과부하) 또는 타임아웃 발생 시 즉시 전환
  - 연동: OpenAI API (Standard Fetch)

**금지 모델**:
- gemini-1.0-*, gemini-1.5-* (종료됨)
- gemini-2.0-* (2026-06-01 종료 예정)

**Fallback 체인**: Gemini(AI) → OpenAI(GPT) → Naver 일반 검색(Local)
- 타임아웃: 20초 (각 단계별 개별 타임아웃 관리)
- 지연 경고 표시: 8초
- Gemini 503 발생 시: 20초 대기 금지, **4초 내외 Fail-Fast 폴백**

### 1-5. 코드 수정 원칙
- 원본 코드의 상수/설정값 임의 변경 금지
- 명시적 지시 없이 "개선" 금지
- 코드 변경과 무관한 파일 건드리기 금지
- 오너가 지시하지 않은 것은 만들지 않는다

### 1-6. 배포 전 필수 검증 (영구)
1. 코드 수정 후 필수: `node -c [파일경로]`로 문법 검사
2. 문법 오류 있으면 push 금지
3. 로컬에서 최소 1회 실행 테스트
4. 위 검증 없이 "완료" 선언 금지
5. Vercel 빌드 Ready = 작동 보장이 아님을 인지

## 2. 운영 규율 (영구 · 이번 세션부터 적용)
==============================================

### 2-1. 완료 보고 금지

**ThisOne이 제대로 완성되기 전까지 "완료했습니다" 보고 금지.**

"완성"의 단일 정의:
- 아래 **섹션 4의 검증 프로토콜을 모두 통과**하고
- 오너가 실제로 확인한 후 **"OK, 통과"라고 명시적으로 승인**한 상태

이 조건 미충족 시 다음 표현 사용 금지:
- ❌ "수정 완료", "개선 완료", "구현 완료", "해결되었습니다"
- ❌ "이제 잘 작동할 것입니다", "문제가 해결될 것입니다"
- ❌ "대략 해결됐습니다"

허용되는 중간 보고:
- ✅ "ranking.js의 RANKING_PROMPT 수정했고, 지금 로컬에서 3개 쿼리로 검증 중"
- ✅ "수정 후 '로보락 S8 MaxV Ultra' 재테스트 결과: 상위 3개 중 본품 0개. 여전히 실패. 다음 시도: {X}"
- ✅ "현재까지 변경한 파일: {목록}. 오너 검증 필요"

### 2-2. 10분 규칙

하나의 문제를 **10분 이상 시도해도 해결되지 않으면 즉시 작업 중단**하고 오너에게 보고.

보고 형식 (고정):
```
[작업 중단 보고]
- 문제: (한 줄 요약)
- 시도한 것: 1) ... 2) ... 3) ...
- 각각의 결과: ...
- 막힌 지점: (정확히 어디서 멈췄는지)
- 추정 원인: (1~2가지)
- 오너에게 필요한 것: (정보/권한/결정)
```

10분 규칙이 적용되는 상황:
- 같은 에러가 3번 이상 반복될 때
- 동일한 파일을 4번 이상 수정해도 증상이 같을 때
- CAPTCHA, 인증, 외부 차단 등 우회 불가능한 장벽을 만났을 때
- 라이브러리/API가 문서대로 동작하지 않을 때
- 무엇을 해야 할지 모르는 상태로 추측으로 코드를 쓰고 있을 때

### 2-3. 추측 금지

금지 패턴:
- ❌ "아마 이 파일일 것입니다" → 확인하고 말해라
- ❌ "이 방식이 맞을 것 같습니다" → 테스트해보고 말해라
- ❌ "일반적으로는..." → 이 프로젝트 실제 코드를 봐라

허용 패턴:
- ✅ "파일을 읽어본 결과 X였습니다"
- ✅ "테스트 실행 결과 Y였습니다"
- ✅ "문서를 확인했는데 Z라고 되어 있습니다"

### 2-4. 집중 우선순위

현재 세션의 지시받은 작업 외 금지:
- UI 개선 제안 (오너가 지시한 부분 외)
- 다른 기능 추가
- 자발적 리팩토링
- "보너스로 이것도 수정했습니다" 류 확장

오너가 명시적으로 지시한 것만 한다.

## 3. 현재 작업 (이번 세션)
==============================================

### 목표: 목록형 레이아웃 전면 재구성

**폐기할 것:**
- 기존 3카드 그리드 컨셉 (DOM + CSS 완전 제거)
- 가성비/신뢰/ThisOne추천을 카드로 분리하는 구조

**새 구조:**

1. 결과 컨테이너
```css
display: flex;
flex-direction: column;
gap: clamp(8px, 2vw, 16px);
width: min(95%, 900px);
margin: 0 auto;
```

2. 각 항목(행) 내부 구조
[썸네일] [정보 영역] [가격/배지]
- 썸네일 폭: `clamp(80px, 15vw, 120px)`
- 정보 영역: `flex: 1`
- 우측에 가격 + 배지

3. 배지 전환
- 가성비 / 신뢰 / ThisOne추천을 각각 구분 색상 배지로
- 제품명 옆 또는 행 우측에 배치

4. 폰트
- `clamp()` 기반 반응형

5. 모바일(~640px)
- 동일 목록 구조 유지
- 썸네일만 축소, 정보 영역 폭 비율 조정
- 카드가 화면 100% 차지하는 세로 스택 금지

### 부수 수정

**브레이크포인트 복원:**
- main.css의 모든 @media 쿼리 확인
- 768px로 된 부분 있으면 전부 640px로 복원

**모바일 검색 후 레이아웃 버그:**
- 증상: 검색창(.search-wrap)이 좌측 세로 쌓임, "비/스/포/크" 글자 세로 배치
- 원인: body.search-mode 상태에서 모바일 레이아웃 규칙 누락
- 수정: 검색창 상단 유지, 결과는 아래로 세로 정렬

**PC 검색 중간 멈춤 버그:**
- 증상: "검색패턴 관찰", "데이터 수집" 후 멈춤 / "검색 중 오류 발생"
- 원인: requestIntentInfer 또는 buildCandidates 실패
- 수정: 모든 외부 API 호출에 Fallback + timeout 적용
- 검증: Gemini 503 에러 발생 시 4초 내외 Fail-Fast 작동 확인

## 4. 검증 프로토콜 (영구 · 절대 어기지 말 것)
==============================================

### 4-1. 검증 엄격 규칙

1. 섹션 2-1의 "완료" 선언 전, **다음을 모두 만족**해야 함:
   - 동일 작업 최소 3회 반복 테스트
   - 3회 모두 동일한 성공 결과
   - Vercel 로그 에러 0개
   - Console 에러 0개

2. 간헐적 성공은 실패로 간주
   - 1회 성공 + 2회 실패 = **실패**
   - 3회 연속 성공해야만 통과

3. 수정 전 반드시:
   - 원인을 구체적으로 특정 (추측 금지)
   - 수정 계획을 텍스트로 명시
   - 오너 확인 후 실행

### 4-2. 필수 검증 항목

**뷰포트 3구간 테스트:**
- 1920px (PC 대형)
- 1366px (노트북)
- 390px (iPhone 12)

**3구간 전부에서 확인:**
- 목록형 "행 나열" 구조 유지
- 좌우 빈 박스 없음
- 가로 스크롤 없음
- 글자 세로 쌓임 없음
- Console 에러 0개

**검색 테스트 (3회 연속 성공 필요):**
- 로보락 S8 MaxV Ultra
- 다이슨 에어랩 멀티 스타일러
- 비스포크 AI 콤보

### 4-3. 완료 보고 제출물 (모두 오너에게)

1. Console 스크린샷 (PC 1920px + 노트북 1366px + 모바일 390px)
2. 검색 결과 스크린샷 (3개 제품 × 3개 뷰포트 = **9장**)
3. 수정한 파일 목록 및 변경 사유
4. Network 탭 실패 요청 리포트
5. AGENTS.md 업데이트 확인 (필요 시)

### 4-4. 배포 흐름

1. 모든 검증 통과 + 제출물 준비
2. 오너에게 "검증 완료, 제출물 첨부" 보고
3. **오너의 명시적 "OK, push해" 지시 대기**
4. 지시 받은 후에만 `git commit + push`
5. `vercel --prod`, `git push main`은 오너 확인 필수

**자동 push 금지. 검증 스킵 절대 금지.**
