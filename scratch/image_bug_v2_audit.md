# 이미지 검색 버그 조사 보고서 (v2) - Intent 분석 결과 오염

**조사 일시**: 2026-04-24
**대상 현상**: 맥미니 이미지에 대해 `intentInfer`가 "아이패드 에어 M2"를 반환하여 잘못된 검색이 수행됨

---

## 1. 로그 및 코드 실행 확인 (Audit Item 1 & 3)

- `js/thisone_app_v3_final.js` 확인 결과, 274행(`[Vision] 1차 이미지 분석 시작...`) 및 279행(`[Vision] 분석 성공: ${finalSearchQuery}`) 로그가 정상적으로 포함되어 있음.
- 오너의 테스트 결과에서 아이패드가 검색된 것으로 보아, `intentProfile.refinedSearchTerm`에 "아이패드 에어 M2"가 담겨 반환되었음이 확실시됨 (그렇지 않았다면 검색 쿼리가 "아이패드"가 될 수 없음).

---

## 2. `intentInfer` 결과 오염 원인 분석 (Audit Item 2)

### [확정 원인: 텍스트 쿼리에 의한 비전 인식 바이어스]
`api/intentInfer.js`의 프롬프트 구성에 치명적인 약점이 발견되었습니다.

- **데이터 주입**: AI에게 `현재 검색어(query)`와 `검색 히스토리(trajectory)`를 이미지와 함께 전달함.
- **AI의 판단 착오**: 사용자가 입력창에 "아이패드 에어 M2"를 남겨둔 상태에서 이미지를 올리면, AI는 사용자의 텍스트 의도를 우선시하거나 이미지와 텍스트를 결합하려 시도함.
- **증상**: 비전 분석으로는 맥미니임을 인지하더라도(Rationale 등에서는 언급), `refinedSearchTerm` 필드에는 사용자가 입력한 "아이패드 에어 M2"를 그대로 적어주는 "확증 편향" 또는 "텍스트 우선 순위" 현상이 발생함.

### [데이터 전달 경로 확인]
- `queryImage`는 `base64` 형태로 정상적으로 Gemini/GPT API에 전달되고 있음 (134-141행).
- 그러나 사용자가 이미지로만 찾고자 할 때도 `queryText`가 "아이패드"로 전달되어 AI를 혼란에 빠뜨리고 있음.

---

## 3. 재현 가능 여부 및 가설

- **재현 시나리오**: 
  1. 입력창에 "아이패드 에어 M2" 입력.
  2. 맥미니 이미지 업로드 후 검색.
  3. `intentInfer`는 `refinedSearchTerm: "아이패드 에어 M2"` 반환. (성공으로 간주됨)
  4. 네이버 검색은 "아이패드"로 수행됨.
  5. 랭킹 AI는 "이미지는 맥미니인데 결과는 아이패드"라며 혼란스러운 설명을 생성함.

---

## 4. 향후 조치 제안 (Phase 2)

- **이미지 검색 시 텍스트 분리**: `queryImage`가 있을 경우, 입력창의 텍스트(`queryText`)를 AI에게 "참고용"이 아닌 "무시 가능성"을 명시하거나, 아예 전달하지 않는 옵션 검토.
- **프롬프트 강화**: "텍스트 쿼리와 이미지 내용이 충돌할 경우, 반드시 이미지 속 제품을 우선하여 검색어를 생성하라"는 강력한 지시문 추가 필요.
- **세션 초기화**: 이미지 검색 시 이전 검색 기록(`trajectory`)이 현재 인식에 방해되지 않도록 가중치를 조정하거나 명확히 구분해야 함.
