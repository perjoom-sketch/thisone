# 이중 AI 호출(Dual Call) 구조 개선 보고서

**기존 문제점**: 
이미지 검색 시 `sendMsg` 함수에서 `requestIntentInfer`를 두 번 호출하고 있었습니다.
1. 1차: 검색 쿼리 생성을 위해 호출 (275행)
2. 2차: 랭킹 및 필터링용 의도 분석을 위해 `Promise.all` 내에서 재호출 (298행)

이로 인해 첫 번째 호출이 실패하고 두 번째 호출이 성공할 경우, "검색 결과는 이전 텍스트 쿼리(아이패드)인데, 분석 결과는 이미지(맥미니)"인 불일치 현상이 발생하여 결과가 0건이 되는 치명적 버그가 유발되었습니다.

**개선 사항**:
- 1차 호출 결과(`intentProfile`)가 존재하면 이를 변수에 보관하고, 2차 호출 시점에서 이를 그대로 재사용하도록 수정했습니다.
- 이를 통해 API 호출 비용을 절감하고, 검색 결과와 필터링 기준 간의 데이터 일관성을 강제했습니다.
- 만약 이미지가 없는 일반 텍스트 검색일 경우에만 필요 시 1회 호출하도록 통합했습니다.

**결과**:
- 데이터 일관성 확보 (이미지 검색 쿼리 불일치 해결)
- API 토큰 소모 감소
- 분석 속도 향상
